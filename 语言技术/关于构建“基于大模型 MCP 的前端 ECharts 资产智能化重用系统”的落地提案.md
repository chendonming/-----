# 关于构建“基于大模型 MCP 的前端 ECharts 资产智能化重用系统”的落地提案

## 一、 背景与业务痛点

经过长期的业务迭代，我们公司内部已经沉淀了大量高质量、符合公司业务规范的 ECharts 图表配置（包含复杂交互、3D立体效果、大屏专属特效等），这些宝贵的资产目前以 JSON 格式存储于公司的 MySQL 数据库中，并配有在线预览功能。

然而，在实际的日常开发中，前端及大屏开发人员仍面临以下痛点：
1. **资产检索困难**：现有的基于关键字（LIKE）的搜索无法理解开发者的“自然语言需求”（例如搜“有没有高端一点的暗黑风格柱子”，传统数据库无法命中）。
2. **“重复造轮子”严重**：由于找不到现成配置，开发者经常向 AI 求助从零手写 ECharts 配置，导致产出的图表风格不统一，且往往达不到公司内部沉淀的高标准。
3. **协同断层**：找到模板 -> 复制 JSON -> 调整代码 -> 查阅文档，整个闭环存在大量的复制粘贴和人工比对工作。

为了彻底盘活这些内部沉淀资产，极大提升前端开发效率，我们计划引入 **MCP（Model Context Protocol，大模型上下文协议）** 与 **向量检索技术**，打造一套智能化的图表生成工作流。

## 二、 核心价值与应用场景体验（我们要做成什么样）

接入该系统后，我们的开发模式将发生质的飞跃。以使用 Cursor/Claude 等支持 MCP 的 AI 编程工具为例：

*   **👨‍💻 程序员在 IDE 中输入指令**：
    > “帮我生成一个带立体效果的柱状图，适合用在汇报大屏上，数据换成我们这个月的营收[10, 20, 30]。”
*   **🤖 AI 自动执行的连贯操作**：
    1. AI 理解需求，自动调用公司内部的 **ECharts MCP 搜索工具**。
    2. 系统通过**语义理解**（向量检索），瞬间在公司数据库中锁定最符合“立体、大屏、柱状图”特征的 3 个内部图表模板。
    3. AI 结合查到的标准 JSON 配置，自动将数据替换为 `[10, 20, 30]`，并直接在 IDE 中生成最终的 Vue/React 代码。
    4. AI 会在对话框附带输出：*“已为您使用公司内部图表库的【3D科技风柱状图】模板，点此链接可在线预览原版效果：`http://demo-url...`”*

**结果**：原本需要耗费 1-2 小时寻找模板、调试配置的工作，被缩短至 **10 秒钟的自然语言对话**，并且产出的代码 100% 符合公司的 UI 规范。

## 三、 技术架构设计（低成本、零侵入方案）

为了保证业务数据库的稳定，并以最快速度验证效果，本方案采用**“业务与 AI 检索解耦”**的架构，并在测试期采用免部署的云端向量数据库。

**架构图解：**
```text
[ 开发者 IDE (Cursor/Claude) ] 
        │
        ▼ (自然语言提问)
[ 本地/内部 MCP Server (Node.js) ]  <--- 核心开发模块
        │
        ├── 1. 将提问转为向量 (对接大模型 Embedding API)
        │
        ├── 2. 向量相似度检索 ---> [ Qdrant Cloud (云端向量数据库) ] 
        │                          (仅存储图表 ID + 语义向量)
        │
        └── 3. 提取完整配置   ---> [ 公司内部 MySQL 数据库 ]
                                   (存储 JSON 配置及 Demo 链接)
```

**为什么选择这套技术栈？**
1. **采用 Qdrant Cloud 向量检索**：相比于传统 SQL，向量数据库能真正理解“高大上”、“科技感”等业务语义。测试期使用 Qdrant Cloud 免费额度，**零服务器成本，零运维压力**。
2. **不污染现有 MySQL 架构**：核心 JSON 数据仍留在原业务库，Qdrant 只做检索目录（类似系统索引），数据极其安全，对现有线上业务**零侵入**。
3. **MCP 协议标准化**：MCP 是目前 AI 行业的标准扩展协议，开发一次，即可无缝接入 Claude Desktop、Cursor、Cline 等所有主流 AI 生产力工具。

## 四、 项目实施路线图 (Phases)

本项目具有“低成本、见效快”的特点，计划分为两个阶段落地：

### 第一阶段：MVP 最小可行性验证（预计 1-2 周）
*   **数据清洗与向量化**：编写 Node.js 脚本，抽取 MySQL 中的图表标题、描述、标签等文本信息，调用大模型生成向量（Embedding），写入 Qdrant Cloud。
*   **开发 MCP Server**：开发并封装内部检索 Tool，实现“输入自然语言 -> 检索向量库 -> 查询 MySQL JSON -> 组装返回结果”的核心逻辑。
*   **小范围试用**：在项目组内部 2-3 名核心前端开发者中配置 Cursor MCP 试用，收集体验反馈并调优 Prompt。

### 第二阶段：内部推广与双向闭环（预计 2 周）
*   **服务内网化部署**：将 MCP Server 部署至公司内网服务器，供全员接入调用。
*   **数据自动同步机制**：在现有的图表管理后台新增逻辑：当员工保存新的图表配置时，自动触发向量化并同步至 Qdrant 库，保证 AI 检索的数据永远是最新的。
*   **支持“反向沉淀”功能（选做）**：为大模型增加 `Save Tool`，开发者在 IDE 中调优好的极品图表，可以通过一句话“帮我存入公司图表库”，直接写入 MySQL。

## 五、 预期收益总结 (ROI)

1. **大幅降低研发成本**：复杂图表的开发时间从“小时级”降至“分钟级”，彻底消灭“找配置、调样式”的垃圾时间。
2. **提升公司资产利用率**：让沉睡在 MySQL 里的高质量配置“活起来”，成为每个开发者的贴身智能助理。
3. **规范研发交付标准**：由于 AI 优先从内部标准库中取用模板，前端界面的风格一致性将得到极大提升。
4. **团队 AI 工程化能力基建**：该项目的落地将跑通“私有数据 + 向量检索 + MCP + IDE”的完整技术栈，未来可以快速复刻到“公司内部组件库检索”、“内部业务代码生成”等更广阔的场景中。
