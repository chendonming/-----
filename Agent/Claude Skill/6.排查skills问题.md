## 排查 Skills 问题

_（4 分钟）_

当 skills 未按预期工作时，问题通常属于几类可预测的情况。本视频将逐一讲解——从 skill 未触发、优先级冲突到运行时失败——并提供一套系统化的排查方法。你还将学习如何使用 skills 验证器工具，以及如何通过 `claude --debug` 诊断加载问题。

#### 核心要点

- 从 **skills 验证器工具** 开始——它能在你花时间排查其他问题前就发现结构错误  
- 如果 skill **未触发**，几乎总是因为描述不够匹配——添加与你实际提问方式一致的触发短语  
- 如果 skill **未加载**，请检查 `SKILL.md` 是否位于命名目录内（而非 skills 根目录），且文件名必须严格为 `SKILL.md`  
- 如果 **错误的 skill 被激活**，说明你的描述过于相似——让它们更明确地区分开来  
- 对于 **运行时错误**，请检查依赖项、文件权限（`chmod +x`）和路径分隔符（所有平台统一使用正斜杠 `/`）

当 skills 出现问题时，通常归为以下几类：未触发、未加载、存在冲突，或运行时失败。好消息是，大多数修复都非常直接。

## 使用 Skills 验证器

首先尝试使用 Agent Skills 验证器命令。安装步骤因操作系统而异，但使用 `uv` 是最快捷的安装方式。

安装完成后，你可以进入 skill 目录运行验证器，也可以从任意位置执行。它能提前捕获结构问题，避免你在其他方面浪费调试时间。

## Skill 未触发

你的 skill 存在且通过了验证，但 Claude 在你预期的时候并未使用它。原因几乎总是 **描述（description）不够匹配**。

Claude 使用语义匹配，因此你的请求必须与描述在语义上有足够重叠。若重叠不足，则无法匹配。解决方法如下：

- 将你的描述与你实际提问的方式进行对比  
- 添加用户**真正会说的触发短语**  
- 用不同表述测试，例如：“帮我分析这段代码性能”、“为什么这么慢？”、“让它跑得更快些”  
- 如果某些变体未能触发 skill，请将这些关键词加入描述中  

## Skill 未加载

如果你向 Claude 询问 “有哪些可用的 skills” 时看不到你的 skill，请检查以下结构要求：

- `SKILL.md` 文件必须位于一个**命名的子目录中**，不能直接放在 skills 根目录下  
- 文件名必须**严格为 `SKILL.md`** —— “SKILL” 全大写，“md” 小写  

运行 `claude --debug` 查看加载错误。留意包含你 skill 名称的日志信息。有时仅凭这条日志就能直接定位问题。

## 错误的 Skill 被激活

如果 Claude 使用了错误的 skill，或在多个 skill 之间显得犹豫不决，很可能是因为它们的描述**过于相似**。请让描述更加明确和差异化。越具体的描述不仅能帮助 Claude 判断何时使用该 skill，还能避免与其他名称相近的 skill 发生冲突。

## Skill 优先级冲突

如果你的个人 skill 被忽略，可能是因为存在同名的**更高优先级 skill**（如企业级 skill）。

![image](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1771527274%2FSkills6_12.1771527274623.png)

例如，如果组织已部署了一个企业级 “code-review” skill，而你也创建了一个同名的个人 skill，那么企业版将始终优先生效。你的选项有：

- 将你的 skill **重命名为更独特的名称**（通常这是最简单的方案）  
- 与管理员沟通企业级 skill 的配置  

## 插件中的 Skills 未显示

安装了插件却看不到其中的 skills？请尝试：**清除缓存、重启 Claude Code、重新安装插件**。

如果问题依旧，可能是插件的目录结构有误。此时，验证器工具的价值就体现出来了。

## 运行时错误

skill 成功加载，但在执行过程中失败。常见原因包括：

- **缺失依赖**：如果你的 skill 使用了外部包，必须确保它们已安装。建议在 skill 描述中注明所需依赖，以便 Claude 知晓环境要求。  
- **权限问题**：脚本需要可执行权限。对 skill 引用的所有脚本运行 `chmod +x`。  
- **路径分隔符**：**所有平台统一使用正斜杠 `/`**，即使在 Windows 上也应如此。

## 快速排查清单

- **未触发？** → 优化描述，添加真实触发短语  
- **未加载？** → 检查路径、文件名和 YAML 语法  
- **错误 skill 被使用？** → 让各 skill 描述彼此区分更明显  
- **被更高优先级 skill 覆盖？** → 检查优先级层级，必要时重命名  
- **插件 skills 缺失？** → 清除缓存并重新安装  
- **运行时失败？** → 检查依赖、权限和路径
