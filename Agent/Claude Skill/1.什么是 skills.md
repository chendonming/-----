## 什么是 skills？

_（3 分钟）_

本视频介绍了 skills —— 可复用的 Markdown 文件，用于教会 Claude Code 自动处理特定任务。与其每次让 Claude 审查 PR 或编写提交信息时都重复说明要求，不如一次性编写一个 skill，Claude 就会在相关任务出现时自动应用它。视频内容涵盖 skills 的定义、存放位置，以及与其他 Claude Code 定制选项的对比。

#### 核心要点

- **Skills 是包含指令的文件夹**，Claude Code 能够发现并使用它们，从而更准确地处理任务。每个 skill 都存放在一个 `SKILL.md` 文件中，其 frontmatter（前置元数据）包含名称和描述。

- **Claude 使用描述来匹配 skills 与用户请求**。当你请求 Claude 执行某项操作时，它会将你的请求与所有可用 skill 的描述进行比对，并激活匹配的 skill。

- **个人 skills** 存放在 `~/.claude/skills` 目录下，在所有项目中通用。**项目级 skills** 则存放在代码仓库根目录下的 `.claude/skills` 中，任何克隆该仓库的人都能获得这些 skills。

- **Skills 按需加载** —— 与 CLAUDE.md（在每次对话中都会加载）或斜杠命令（需要显式调用）不同，skills 在 Claude 识别到相关场景时会自动激活。

- 如果你发现自己**反复向 Claude 解释同一件事**，那就说明这里应该写一个 skill。

每次你向 Claude 解释团队的编码规范，其实都是在重复劳动。每次 PR 审查，你都要重新描述期望的反馈格式；每次写提交信息，你都要提醒 Claude 你偏好的格式。Skills 正是为解决这个问题而生。

一个 skill 就是一个 Markdown 文件，它一次性教会 Claude 如何完成某件事。之后，只要场景相关，Claude 就会自动应用这些知识。

## Skills 的本质

Skills 是包含指令和资源的文件夹，Claude Code 能够发现并利用它们，以更准确地处理任务。每个 skill 都存放在一个 `SKILL.md` 文件中，其 frontmatter 包含名称和描述。

![image](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1771527257%2FSkills1_05.1771527257795.png)

Claude 正是通过描述来决定是否使用某个 skill。例如，当你请求 Claude 审查 PR 时，它会将你的请求与所有可用 skill 的描述进行匹配，并找到最相关的那个。Claude 会读取你的请求，将其与所有 skill 描述进行比对，并激活匹配的 skill。

下面是一个 skill 的 frontmatter 示例：

```
---
name: pr-review
description: 审查 pull requests 的代码质量。在审查 PR 或检查代码变更时使用。
---
```

在 frontmatter 之下，你可以编写具体的指令——比如你的审查清单、格式偏好，或者 Claude 完成该任务所需的任何信息。

## Skills 的存放位置

你可以根据使用范围将 skills 存放在不同位置：

- **个人 skills** 存放在 `~/.claude/skills`（即你的主目录）。这些 skills 会跟随你应用于所有项目——比如你偏好的提交信息风格、文档格式，或你希望代码被解释的方式。

- **项目级 skills** 存放在代码仓库根目录下的 `.claude/skills` 中。任何克隆该仓库的人都会自动获得这些 skills。团队标准（如公司品牌指南、Web 设计中偏好的字体和配色方案）就应放在这里。

在 Windows 系统上，个人 skills 存放在 `C:/Users/<your-user>/.claude/skills`。

项目级 skills 会随代码一起提交到版本控制系统中，因此整个团队都能共享。

## Skills 与 CLAUDE.md、斜杠命令的对比

Claude Code 提供了多种行为定制方式。Skills 的独特之处在于它们是自动触发且面向特定任务的。以下是三者的对比：

- **CLAUDE.md** 文件会在每次对话中加载。如果你希望 Claude 始终启用 TypeScript 的 strict 模式，这类全局配置应放在 CLAUDE.md 中。

- **Skills** 仅在匹配用户请求时按需加载。Claude 最初只加载 skill 的名称和描述，因此不会占用整个上下文窗口。例如，当你在调试时，PR 审查清单无需加载；只有当你明确请求审查时，它才会被载入。

- **斜杠命令（Slash commands）** 需要你显式输入才能触发，而 skills 不需要。Claude 会在识别到相关场景时自动应用它们。

当 Claude 将某个 skill 与你的请求匹配成功时，你会在终端中看到它被加载：

## 何时使用 Skills

Skills 最适用于那些针对特定任务的专门知识，例如：

- 团队遵循的代码审查标准  
- 你偏好的提交信息格式  
- 组织的品牌指南  
- 特定类型文档的模板  
- 针对特定框架的调试检查清单  

简单判断标准：**如果你发现自己反复向 Claude 解释同一件事，那就应该把它写成一个 skill。**
